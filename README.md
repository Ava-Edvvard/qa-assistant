# QA-Assistant

**QA-Assistant** — это интеллектуальный инструмент для тестировщиков, помогающий в проектировании и актуализации тест-планов. Он анализирует требования (текст + Excel + изображения), выявляет логические нестыковки с помощью ИИ, строит матрицу трассируемости и генерирует детальные тест-сценарии.

## Стек технологий

*   **Backend**: Python, FastAPI, Pydantic, Pandas (парсинг Excel), OpenAI API.
*   **Frontend**: React (Vite), TypeScript, Vanilla CSS (Premium Glassmorphism), Lucide Icons, Axios.
*   **Документация**: Подробное описание в `.md` файлах в папке `/docs`.

## Инструкция по запуску

### 1. Запуск Backend (FastAPI)

Перейдите в папку `backend`, установите зависимости и запустите сервер:

```bash
cd backend
python -m venv venv
# Активация на Windows:
.\venv\Scripts\activate

# Установка зависимостей:
pip install -r requirements.txt

# Укажите ваш OpenAI API ключ в файле .env (по умолчанию работает мок-режим):
# OPENAI_API_KEY=your-real-key

# Запуск сервера:
uvicorn app.main:app --reload --port 8000
```
Сервер будет доступен по адресу: [http://127.0.0.1:8000](http://127.0.0.1:8000)  
Интерактивная документация API Swagger: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

### 2. Запуск Frontend (React)

Перейдите в папку `frontend`, установите зависимости и запустите Vite dev-сервер:

```bash
cd frontend
npm install
npm run dev
```
Интерфейс приложения откроется по адресу: [http://127.0.0.1:5173](http://127.0.0.1:5173)

## Режимы работы

1.  **Проработка нового тест-дизайна** (5 этапов):
    *   Ввод требований и файлов (Excel/изображения)
    *   Корректировка требований в таблице матрицы
    *   Ответы на уточняющие вопросы от ИИ
    *   Редактирование сгенерированных тест-сценариев
    *   Экспорт заполненной матрицы трассируемости и тест-кейсов в Markdown
2.  **Доработка существующего тест-дизайна** (6 этапов):
    *   Добавляется этап сравнения (Diff Report) старых тест-сценариев с новыми.
